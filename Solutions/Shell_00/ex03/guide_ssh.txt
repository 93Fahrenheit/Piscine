------------------------------------------------------------
GUIDE : LA CLÉ SSH - CONCEPT ET COMMANDES UTILES
------------------------------------------------------------

1. 🔐 Qu'est-ce qu'une clé SSH ?
------------------------------------------------------------
SSH signifie "Secure Shell". C'est un protocole qui permet de se connecter
de manière sécurisée à une machine distante via le réseau (souvent un serveur Linux).

Une clé SSH est un système d’authentification basé sur une paire :
- ✨ Clé publique : à partager avec le serveur distant (ex : GitHub)
- 🔒 Clé privée : à garder strictement sur ta machine

Fonctionnement :
- Le serveur compare ta clé publique avec ta clé privée locale
- Si elles correspondent : la connexion est autorisée sans mot de passe

------------------------------------------------------------

2. 📁 Où sont stockées les clés SSH ?
------------------------------------------------------------
Par défaut dans le dossier caché ~/.ssh/

- id_rsa ou id_ed25519         → clé privée (ne jamais la partager)
- id_rsa.pub ou id_ed25519.pub → clé publique (à copier sur les serveurs)

------------------------------------------------------------

3. ⚙️ Créer une paire de clés SSH
------------------------------------------------------------
Commande :
    ssh-keygen -t ed25519 -C "ton@email.com"

Options :
- -t ed25519 : type de clé recommandé (plus rapide et moderne que RSA)
- -C        : commentaire pour identifier la clé

Appuie sur Entrée pour accepter l’emplacement par défaut (~/.ssh/id_ed25519)

------------------------------------------------------------

4. 📤 Ajouter la clé publique à GitHub (ou autre service)
------------------------------------------------------------
Afficher la clé :
    cat ~/.ssh/id_ed25519.pub

→ Copie-colle le contenu dans ton compte GitHub :
    GitHub > Settings > SSH and GPG Keys > New SSH Key

------------------------------------------------------------

5. ⚡ Utiliser la clé SSH avec Git
------------------------------------------------------------
Si ton dépôt est en HTTPS, passe-le en SSH :

    git remote set-url origin git@github.com:username/nom-du-repo.git

→ Tu pourras faire `git push`, `git pull` sans mot de passe

------------------------------------------------------------

6. 🛠️ Commandes utiles
------------------------------------------------------------
Vérifier si des clés existent :
    ls ~/.ssh

Ajouter ta clé à l’agent SSH :
    ssh-add ~/.ssh/id_ed25519

Tester la connexion avec GitHub :
    ssh -T git@github.com

Gérer tes clés en ligne :
    https://github.com/settings/keys

------------------------------------------------------------

7. 🚫 À ne jamais faire
------------------------------------------------------------
- ❌ Ne jamais partager ta clé privée (`id_ed25519`)
- ❌ Ne jamais la committer dans un dépôt Git
- ❌ Ne pas supprimer le dossier `.ssh/` sans savoir ce que tu fais
