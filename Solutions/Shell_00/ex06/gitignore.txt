# FICHE - GÉRER LES FICHIERS IGNORÉS AVEC GIT

🎯 Objectif : Empêcher Git de suivre certains fichiers inutiles (temporaire, config locale…)

────────────────────────────────────────────
1. FICHIER .gitignore
────────────────────────────────────────────

➤ À placer à la racine du dépôt (à côté de .git)
➤ Chaque ligne décrit un motif à ignorer

📌 Exemples :
# Fichiers temporaires
*.tmp
*.swp
*~

# MacOS
.DS_Store

# VS Code
.vscode/

# Binaires
a.out
*.o
*.exe

🛑 Ne PAS écrire de commentaire sur la même ligne qu’un motif :
❌ *.tmp   # fichier temporaire → Git ignore toute la ligne
✅ # fichier temporaire
   *.tmp

────────────────────────────────────────────
2. VÉRIFIER SI UN FICHIER EST IGNORÉ
────────────────────────────────────────────

git check-ignore -v chemin/vers/fichier

→ Affiche la règle qui l’ignore (ou rien s’il n’est pas ignoré)

────────────────────────────────────────────
3. VOIR TOUS LES FICHIERS IGNORÉS PRÉSENTS
────────────────────────────────────────────

Commande :
git ls-files --others --ignored --exclude-standard

📘 Explication de chaque option :

| Option              | Rôle                                                                 |
|---------------------|----------------------------------------------------------------------|
| git ls-files        | Affiche les fichiers indexés et non indexés dans Git                |
| --others            | Montre les fichiers non suivis par Git                              |
| --ignored           | Restreint la liste aux fichiers ignorés                             |
| --exclude-standard  | Utilise les règles standards (.gitignore, .git/info, global ignore) |

📌 Cette combinaison permet d’obtenir la **liste des fichiers ignorés présents sur ton disque**, mais **non suivis par Git**.

────────────────────────────────────────────
4. ARRÊTER DE SUIVRE UN FICHIER DÉJÀ TRACKÉ
────────────────────────────────────────────

git rm --cached nom_du_fichier

✔ Retire le fichier de Git mais le laisse sur ton disque

────────────────────────────────────────────
5. RAPPEL IMPORTANT
────────────────────────────────────────────

❗ Git ignore uniquement les fichiers qui ne sont **pas déjà suivis**.

✔ `.gitignore` fonctionne seulement si :
- le fichier concerné n’a jamais été ajouté (git add)
- le dossier qui contient le fichier est bien suivi par Git

────────────────────────────────────────────
6. POUR ALLER PLUS LOIN
────────────────────────────────────────────

man gitignore       ← règles d’écriture
man git-ls-files    ← lister les fichiers Git
git status --ignored
